Robobench: A Comprehensive Evaluation Benchmark for Multimodal Large Language Models as Embodied Brain

RoboBench is a comprehensive benchmark for evaluating MLLMs as embodied brains, encompassing 5 dimensions, 16 capabilities, and 26 tasks. It reflects the key stages of embodied reasoning: understanding explicit and implicit instructions, perceiving embodiment, objects, and scenes, planning long-horizon tasks through subtask decomposition, generating spatially aligned affordance to support subtask planning, and performing error analysis with reflective correction (top). Each dimension is illustrated with a demo (bottom), and task types are color-coded to showcase the diversity and complexity of RoboBench.

Abstract

Multi-modal Large Language Models (MLLMs) are emerging as powerful candidates for the “brain” of embodied agents. However, existing benchmarks fail to capture the full spectrum of capabilities required for embodied tasks, including instruction comprehension, perceptual reasoning, generalizable planning, affordance prediction, and error analysis. To bridge these gaps, we present RoboBench, a large-scale benchmark that systematically evaluates MLLMs as embodied brains across 5 core dimensions, 15 capabilities, and 26 task types. It incorporates specially designed tasks such as cross-embodiment, cross-object, and cross-view planning, together with static and dynamic affordance reasoning, for fine-grained and realistic assessment. To ensure comprehensive and robust planning evaluation, we propose a DAG-based LLM evaluation framework that accounts for skill selection, object identification, task execution order variation, and task completion quality. Extensive evaluations of closed-source and open-source MLLMs reveal critical limitations overlooked by previous benchmarks. Notably, current models struggle with understanding implicit demand instructions, cross-embodiment planning, object interaction affordance reasoning, and execution error analysis. RoboBench provides a rigorous, diverse, and realistic evaluation protocol to guide the development of next-generation MLLMs with stronger embodied intelligence.

Dataset Construction Pipeline

RoboBench combines data from recent open-source large-scale datasets and our self-collected real-world datasets. We design dedicated workflows for different task types, color-coded as green for Perception, blue for Planning, and red for Reflection. The pipeline involves preprocessing, tool-assisted human annotation, and metadata standardization. 1) Perception: Tools such as VLM captioning or detection models provide preliminary labels, which are refined by human annotators. 2) Planning: VLMs generate task summaries and segmented captions; these are first converted into predefined function calls for evaluation, then processed by LLMs to extract key information for task categorization. Implicit goal planning tasks are derived by rephrasing explicit task summaries into implicit demand expressions. 3) Reflection: Low-level errors are identified from failed execution videos and manually annotated, while high-level errors are created by modifying instructions in correct cases to form error pairs. The resulting metadata are used to automatically generate Q&A items in standardized binary classification, multiple choice, and multi-step multiple choice formats, ensuring compatibility with both open and closed-source MLLM models.
